Deployment

This should have been the most exciting time for us - our application was fully completed all we need to do now is upload it and that is it! But alas no - if only it was that easy... I bought hosting with a company called Dreamhost who were crazy cheap with crazy bandwidth and they said they supported Ruby on Rails - woohoo! I googled around a bit for tutorials on deploying a app on Dreamhost and came up with a lot of hate mail saying that DH is a pain in the ass to deploy on - it is possible but stay away if you have the cash (which I did not)... So with no other option I read the 15 tutorials on how to deploy the bugger, spent countless hours reading the production.log file and after two days of swearing finally got it working. I was having some unusual errors so there was not a lot of support on forums and the likes.

But it is up and running now and that is all that matters. Again this is just one of the great experiences that I have had whilst developing this application - I really do know now what is involved in creating something from start to finish.

Issues

Issues are just things that you have to deal with I have found out - they come and go but they are always there. These are just some of the issues that I had to get my head around while coding for mnf.b:

  • Deployment
  • Hosting
  • Databases
  • Database Users / Permissions

Upon reflecting on these issues that I had to figure out solutions for, it has become evident that no project is going to be without them. I think I came into this project thinking the hardest part was going to be learning a new language - I wasn't aware that all these other issues existed. I am glad that I came across them though - from what I hear I will be encountering them from now until forever.
